Key observations

  • With an increase in penalty values, we can clearly see that the number of lines in our optimal solution will decrease. This is because the algorithm now finds it more cost effective to fit more number of points to each line rather than create a new line completely.
  • The dynamic programming approach has reduced what would have been an exponential time complexity problem to a polynomial one.
  • This solution allows for a much better loss function value than just a single line.
  • Dynamic programming is possible in this case due to the fact that we can get the answer for the current state by utilizing the answer from some previous state.

Time Complxity Analysis

  • The bottleneck for the time complexity in this algorithm is the dynamic programming. For each pair of indices i and j, we find the line of best fit between them. Finding the line of best fit takes O(n) worst case, and since there are O(n^2) pairs of indices, the total complexity is O(n^3).

Space Complxity Analysis

  • The bottleneck for the space complexity in this algorithm is storing the dynamic programming states. We only store the best answer up till index i, hence we only need O(1) memory for each i. This totals up to an overall memory complexity of O(n).

Visualizer Documentation

  • The visualizer was implemented using matplotlib library in python. First the C++ program writes the necessary details into a temporary file called data.txt.
  • The python code reads from that file. It first reads the number of points, as well as the coordinates of those points. It then plots the points on the grid.
  • It then reads the number of line segments present in the optimal solution. After this, it also reads the start point and the end point for each of these line segments. It plots the line segment exactly between these two points.
  • The visualizer also prints the minimum possible cost on top of the grid. Finally it closes the grid. More details can be seen from the code and the comments present in it.
