Key observations

  • One key observation we make here is that as the number of rectangles increases, The shape of the Contour starts to resemble that of a perfect rectangle. This holds in accordance with probabilistic principles since the rectangles are randomly generated
  • We also see that with an increase in the number of rectangles, the measure and contour are approximately proportional by a power of 2, i.e measure is approximately proportional to contour length squared. This is as expected since contour is nothing but perimeter and measure is area.
  • Since the algorithm is of the complexity O(nlogn) where n is the number of rectangles, it runs relatively quickly even for a larger number of rectangles

General Discussion on the Algorithm

  • The important functions were implemented exactly as described in the paper. More specifically, RECTANGLE_DAC, STRIPES, copy, blacken, concat, contour,etc. Some functions however were left unexplained by the author, such as finding the intersection of two sets of intervals or finding the free X Intervals of a stripe. To maintain the complexity O(nlogn) we needed these functions to run in O(n) time. The algorithms we used for some of these functions are written below:
  • Interval Intersection: To find the intersection between two sets of intervals, we indirectly used a hash function to keep track of what kind of intervals were present in the first set and simply check if that interval was there in the second set. Since the checking operation is O(1) in most cases, the overall complexity is O(n).
  • Free Query: To find the free query of a stripe, we ran through all the leaf nodes in the ctree of the stripe and simply used a few conditionals to check which intervals were part of the free query. The complexity of this part was O(h+alpha) where h is the height of the ctree and alpha is the number of free intervals. This is the same complexity as expected by the paper.
  • Finding subsets and supersets: Some of the functions required for to find subsets and supersets of intervals. The brute force method for this would require an O(n^2) time complexity, but this can be brought down to O(n) with an optimized two pointer algorithm which is what we implemented. Hence even this function was O(n).
  • For measure, the major functions like copy,blacken and concat described by the paper are already proved to be O(n) each. Hence the recursion formula would be of that form T(n) = 2*T(n/2) + O(n) + O(1). This is well known to reduce to O(nlogn).
  • For Contour, it is similar to measure in all but one place: finding the free intervals. The computation of stripes itself will be the same so the total complexity. For each free interval query, (since the height of the tree is logn), the complexity will be O(logn + alpha). Hence the total complexity would be O(nlogn + p) where p is the number of contour pieces.
  • In both cases, the space complexity is O(nlogn) calculated similar to the time complexity. Contours can be calculated in O(p) time and space where p is the number of contour pieces.
